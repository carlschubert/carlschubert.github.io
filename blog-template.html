    Big O Notation

    Big O notation is used to describe the complexity of an algorithm. Simply put it uses the worst-case senerio to estimate things like mermorty use and execution time.

    O(1) decribes an algorithm that will always use the same amount of memory no matter what the input.

    O(N) decribes an algorithm that performance degrades linearly as input increases. Tasks like a for loop fall into this category as the more elements are iterated the more resourses are required.

    O(N^2) are algorithims that performance degrades proportional to the square of the input. This can be common with iterations ofver nested data sets. Algorithms can also be O(2^N) in which each additional element doubles the performance requirments!
